# Apocalypse Now 3.0.5 — Reconstructed & Enhanced

Неофициальная, максимально восстановленная и доработанная версия **Apocalypse Now — Post Apocalyptic Arsenal** для Minecraft 1.20.1 Forge.

Исходный мод и весь его оригинальный контент созданы **JPZamps**. Этот репозиторий не является официальным репозиторием автора и не выдаёт восстановленную версию за официальный выпуск.

- [Оригинальный выпуск 3.0.5 для Forge 1.20.1](https://www.curseforge.com/minecraft/mc-mods/apocalypse-now/files/6364603)
- [Профиль JPZamps на CurseForge](https://www.curseforge.com/members/jpzamps/projects)

## О проекте

Проект восстановлен из официального JAR `apocalypsenow-3.0.5-forge-1.20.1.jar` с использованием официальных Mojang mappings. Сохранены исходные `modid`, версия и идентификаторы контента, чтобы сборка оставалась совместимой с существующими мирами и сборками на базе версии 3.0.5.

Техническая основа:

- Minecraft 1.20.1;
- Forge 47.4.10;
- Java 17;
- Curios 5.12.1+1.20.1;
- `modid`: `apocalypsenow`;
- версия мода: `3.0.5`.

## Что исправлено

- высокие и широкие блоки работают как единый объект: корректные коллизия, выделение, взаимодействие, разрушение, инструмент, звук и частицы;
- исправлены области взаимодействия тел, капканов, баррикад, мебели и других выступающих моделей;
- аксессуары Curios надеваются правой кнопкой мыши и безопасно меняются местами;
- второй слой скина скрывается под соответствующими частями брони;
- исправлены выбор целей, ответная агрессия, дистанция удара, фракционная логика и условия естественного появления мобов;
- ходячие заражённые, бегуны и орды получили ванильную боевую анимацию рук зомби;
- исправлены укусы, инфекция и зомбификация без накопления тысяч отложенных задач;
- устранены двойное выполнение, неправильное расходование предметов и клиент-серверные рассинхронизации в медицине, жидкостях, книгах, радио, взрывчатке и инструментах;
- исправлены бензопила, открывалка, удобрение, перцовый баллончик, набор сапёра, гранаты, бочки и сборщики дождевой воды;
- исправлены неработающие рецепты и повреждённые ссылки моделей, текстур, тегов и лут-таблиц;
- творческие вкладки упорядочены по назначению, семействам и типам предметов;
- добавлен полный русский перевод: 1569 из 1569 ключей;
- удалены бесполезные вечные тики и устаревшие ресурсы, которые Forge не использовал.

Подробный список находится в [CHANGELOG.md](CHANGELOG.md).

## Установка готового JAR

1. Установите Minecraft 1.20.1 и Forge 47.4.10.
2. Установите Curios для Minecraft 1.20.1.
3. Удалите оригинальный JAR Apocalypse Now 3.0.5 из папки `mods`.
4. Поместите собранный `apocalypsenow-3.0.5-forge-1.20.1.jar` в папку `mods`.
5. Не устанавливайте оригинальную и восстановленную версии одновременно: у них одинаковый `modid`.

Перед первым запуском существующего мира рекомендуется сделать резервную копию. Уже установленную высокую или широкую мебель нужно один раз сломать и поставить заново, чтобы создать новые невидимые области взаимодействия.

## Как собрать мод из исходников

Ни Forge MDK, ни отдельный Gradle устанавливать не требуется: всё необходимое для запуска Gradle уже находится в репозитории. При первой сборке потребуется интернет для загрузки Minecraft Forge, mappings и Curios.

### Windows — самый простой способ

1. Установите [JDK 17](https://adoptium.net/temurin/releases/?version=17). Нужна именно **JDK**, а не только JRE.
2. Скачайте репозиторий через **Code → Download ZIP** и распакуйте его либо клонируйте через Git.
3. Откройте папку проекта, в которой лежит файл `gradlew.bat`.
4. В Windows 11 нажмите правой кнопкой мыши по свободному месту папки и выберите **Открыть в терминале**. Другой способ: щёлкните адресную строку Проводника, введите `powershell` и нажмите Enter.
5. В открывшееся окно PowerShell вставьте команду:

```powershell
.\gradlew.bat clean build --no-daemon
```

Команду не нужно добавлять в Java-код или какой-либо файл: она выполняется только в терминале из корневой папки проекта.

После успешной сборки готовый мод появится здесь:

```text
build\libs\apocalypsenow-3.0.5-forge-1.20.1.jar
```

Если команда сообщает, что Java не найдена, перезапустите терминал после установки JDK 17 и проверьте:

```powershell
java -version
```

В выводе должна быть версия Java 17.

### Linux и macOS

```bash
chmod +x gradlew
./gradlew clean build --no-daemon
```

Готовый JAR будет находиться в `build/libs/`.

### IntelliJ IDEA

Откройте `build.gradle` как Gradle-проект, выберите JDK 17 и дождитесь синхронизации зависимостей. Для обычной сборки всё равно достаточно задачи `build` или команды Gradle Wrapper выше.

## Проверка проекта

Полная локальная проверка:

```powershell
.\gradlew.bat clean build --no-daemon
python .\tools\audit_mod.py
```

Аудит проверяет реестры, рецепты, модели, текстуры, языковые ключи, меню, теги, лут-таблицы и ссылки между ресурсами. GitHub Actions также собирает проект при каждом push и pull request.

## Совместимость с существующими мирами

Сохранены `modid=apocalypsenow`, версия `3.0.5` и старые идентификаторы блоков, предметов, сущностей, эффектов и меню. Исправленная сборка предназначена для прямой замены оригинального JAR в сборке Minecraft 1.20.1 Forge.

Для выделенного сервера одинаковый JAR должен быть установлен и на сервере, и у игроков. Для отката после сохранения мира восстановите резервную копию и исходный JAR.
